## Description

MC7 introduces students to the design of hybrid human–AI coordination systems within creative and cultural industries contexts. Using Strategyzer tools as a strategic framework and Claude Skills as an operational infrastructure layer, students prototype AI-augmented practice systems that will later become components of the capstone boundary object in MC9.

The central concern of MC7 is not tool proficiency but **systems thinking**: how do AI tools mediate value flows between activity systems? How do governance structures protect community trust while enabling institutional participation? How do you stress-test a system architecture before live deployment?

MC7 prepares students for the MC9 capstone — a live formative intervention centered on a fundraising event designed as a boundary object between a cause-driven organization and a sponsor/donor ecosystem. MC7 focuses on **designing the system**. MC9 focuses on **defending the system**.

The ClearLED B2B studio incubation at Multimodal Media Lab (Vancouver) serves as the primary guided case study — a live, real-world example of an AI-augmented practice system being built from the ground up, with all the complexity of client relationships, funding constraints, governance questions, and talent sourcing intact.

---

## Learning Outcomes

By the end of MC7, students will be able to:

- [ ] Apply Strategyzer's Business Model Canvas and Value Proposition Canvas to map the structure of an AI-augmented design practice or coordination system
- [ ] Design hybrid human–AI coordination systems that mediate value flows between two or more distinct activity systems
- [ ] Evaluate system designs for feasibility, viability, and desirability using AI-assisted analysis tools
- [ ] Anticipate governance implications of AI-augmented systems — including data governance, community trust safeguards, and IP structuring
- [ ] Model experience flows and feedback loop architecture for digitally mediated cultural events or activations
- [ ] Prototype at least one MC7 system output that will feed directly into the MC9 capstone boundary object
- [ ] Analyze the ClearLED/MML studio incubation as a guided case study, extracting transferable principles for AI-augmented practice formation
- [ ] Produce a self-directed case study applying the same methodology to a systems design context of the student's own choosing

## Relationship to MC9 Capstone

MC7 outputs are not standalone deliverables — they are **prototypes for the capstone**. Examples of MC7 outputs that feed directly into the MC9 boundary object include:

- AI-assisted sponsor visibility logic
- Fundraising engagement optimization models
- Data governance frameworks
- Community trust safeguards
- Experience flow modeling
- Feedback loop architecture
- Governance and IP structuring for a designed live event

Students should treat MC7 as the design and prototyping phase of a system they will later deploy and defend in MC9.

---

## Theoretical Framework

MC7 is grounded in the following frameworks, which students will encounter explicitly:

- **Activity Theory** — modeling activity systems, identifying structural contradictions
- **Change Laboratory methodology** — Strategyzer tools used as second stimulus
- **Boundary object theory** — the fundraiser event as a shared artifact coordinating systems with different institutional logics
- **Formative intervention** — design as systemic change, not just problem-solving

These frameworks are introduced in MC7 and fully applied in MC9.
