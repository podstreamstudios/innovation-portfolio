## Course Description

This microcredential centers on culture as a system, emphasizing spatialization of meaning, curatorial authority, and experiential design as cultural argument. Working in remote teams, students analyze cultural icons as systems (not isolated objects) and design spatial or experiential environments that translate cultural meaning into coordinated encounters.

**The course uses ARIES (Autopoietic Remix for Intertextual Ecological Systems) methodology**, which coordinates cultural production across three layers:
1. **Cultural Reservoir** (Intertextual Inputs) - canonical texts, practices, iconographies
2. **Experiential Assembly** (Remix Operations) - spatial, temporal, atmospheric orchestration
3. **Value Objectification** (Authorization Layer) - licenses, permissions, revenue models

---

## Historical Context: The Time Capsule Frame

This microcredential is derived from a course taught 2002-2006 at SFU's School of Interactive Arts & Technology, **frozen at 2006** just before three fundamental ruptures restructured cultural production:

1. **Podcasting** → Platform-mediated identity
2. **Social media** → Ubiquitous algorithmic mediation
3. **Artificial Intelligence** → Synthetic/generative media

**This positioning is intentional**. The course operates as a **methodological time capsule**, allowing reflexive analysis of how these technologies changed cultural authority, circulation, and production.

## Pre-Visualization Tools as Tertiary Artifacts

### Theoretical Framing

This microcredential treats pre-visualization tools as **Wartofsky TERTIARY artifacts**—not neutral documentation methods but **tools that shape design thinking itself**.

**Wartofsky's three levels of artifacts:**

**PRIMARY (Resources):** 
- Cultural building blocks (films, music, cuisine traditions)
- Physical spaces (ballroom dimensions, AV equipment, catering facilities)
- Budget constraints (licensing costs, staffing expenses, material resources)

**SECONDARY (Processes):**
- ARIES methodology (Cultural Reservoir → Experiential Assembly → Value Objectification)
- Event planning workflows (research → design → budget → pitch)
- Team collaboration protocols (synchronous meetings, asynchronous work, documentation standards)

**TERTIARY (Perspectives/Representations):**
- **Pre-visualization tools = ways of SEEING and ORGANIZING design**
- Tools that reorganize perception of how events will work
- Not just documentation but **imagination tools** that shape what designers think is possible

### Why Pre-Visualization Tools Are Tertiary (Not Secondary)

**If tools were SECONDARY (processes):**
- They would be neutral methods for executing already-made design decisions
- Tool choice wouldn't affect WHAT gets designed, only HOW it's documented
- Designers would think first, then use tools to capture thinking

**Because tools are TERTIARY (perspectives):**
- **Tools shape design thinking before decisions are made**
- Different tools make different design possibilities visible/invisible
- Tool choice determines what designers notice, prioritize, and communicate
- **"When you only have a hammer, everything looks like a nail"** applies to pre-viz tools

**Examples of how different tools shape thinking:**

**Spatial collaboration platforms** (Miro, Figma, Mural):
- Make connections, relationships, systems thinking visible
- Designers using these tools tend to think in networks, flows, coordinated elements
- ARIES layers become interconnected boards with visual relationships

**Presentation software** (PowerPoint, Keynote, Google Slides):
- Make sequential narrative, story arcs visible
- Designers using these tools tend to think in linear progression, dramatic reveals, chapter structure
- ARIES layers become slides with beginning → middle → end flow

**3D rendering tools** (SketchUp, Blender, Unreal Engine):
- Make atmospheric immersion, sensory experience, spatial depth visible
- Designers using these tools tend to think in guest sightlines, lighting effects, material textures
- ARIES layers become walkthrough experiences showing how space transforms

**Spreadsheet/data tools** (Excel, Google Sheets, Airtable):
- Make financial models, resource allocation, cost-benefit analysis visible
- Designers using these tools tend to think in viability, margins, break-even points
- ARIES layers become quantified with every element having associated costs/revenues

**Timeline/project management tools** (Asana, Monday, Gantt charts):
- Make temporal sequencing, dependencies, critical paths visible
- Designers using these tools tend to think in milestones, deadlines, resource scheduling
- ARIES layers become phased implementation with clear start/end dates

### Tool Choice = Value Choice

**Pre-visualization tools embed priorities:**

A design created in **3D rendering + mood boards** prioritizes:
- ✓ Atmospheric immersion (can guests FEEL the transformation?)
- ✓ Sensory coordination (how do lighting, sound, scent layer?)
- ⚠ May under-emphasize financial viability (rendering doesn't show profit margins)

A design created in **spreadsheets + Gantt charts** prioritizes:
- ✓ Financial feasibility (does the budget work?)
- ✓ Operational logistics (can we execute on schedule?)
- ⚠ May under-emphasize guest experience quality (numbers don't show atmosphere)

**ARIES methodology requires BOTH:**
- Experiential quality (Layer 2 - how guests experience the event)
- Financial viability (Layer 3 - how the event sustains economically)

**Therefore, good pre-viz tools must make BOTH visible simultaneously.**

### Why This Matters for Hospitality Careers

**Entry-level event coordinators encounter different tools at different properties:**
- Marriott might use proprietary event planning software
- Independent hotels might use Google Workspace + PowerPoint
- Conference centers might use specialized venue management systems

**Graduates need:**
- Not tool-specific training (tools change constantly)
- But **tool literacy** (how to assess and master new tools quickly)
- Understanding that **tools are perspectives** (not neutral documentation)

**This microcredential teaches transferable visualization intelligence:**
- How to recognize what any tool privileges/obscures
- How to coordinate across multiple tools (no single tool shows everything)
- How to produce client-facing quality regardless of platform

### Connection to ARIES Methodology

**ARIES three layers require different visualization approaches:**

**Layer 1: Cultural Reservoir**
- Best visualized through: Image boards, reference libraries, research documentation
- Tools that show: Connections, influences, intertextual relationships
- Why: Cultural inputs are networked, not linear

**Layer 2: Experiential Assembly**
- Best visualized through: Floor plans, timelines, sensory mood boards, 3D renderings
- Tools that show: Spatial coordination, temporal flow, atmospheric layering
- Why: Experience is embodied, multi-sensory, sequential

**Layer 3: Value Objectification**
- Best visualized through: Budget spreadsheets, revenue models, authorization matrices, Gantt charts
- Tools that show: Financial viability, licensing requirements, operational feasibility
- Why: Value must be quantified, risks assessed, permissions secured

**No single tool does all three well.**

**Professional event design requires tool coordination:**
- Use spatial tools for experiential design (Layer 2)
- Use financial tools for viability analysis (Layer 3)
- Use research tools for cultural documentation (Layer 1)
- Then integrate into unified client presentation

**This is what students practice in MC2:**
- Working across tool ecosystems
- Maintaining ARIES coordination despite using different visualization methods
- Producing unified deliverables that show all three layers clearly

## The Pedagogical Structure: Casablanca @ The American (Exemplar) → Student Applications

### Casablanca @ The American Pre-Visualization (Exemplar)

**Function**: Complete teaching artifact showing HOW ARIES works in practice

**Students analyze the Casablanca @ The American pre-viz PDF to understand**:
- How film iconography (Cultural Reservoir) becomes event design for a specific venue
- How iconic scenes translate to spatial moments within The American's architecture
- How licensing/permissions shape feasibility (Value Objectification)
- How venue identity (American bar culture, Main Street context) coordinates with film mythology

**The American as venue context**:
- **Location**: Main Street, Vancouver (historic entertainment district)
- **Venue identity**: Classic American bar/restaurant with theatrical atmosphere
- **Architectural affordances**: Stage area, bar as focal point, intimate dining spaces
- **Cultural positioning**: Mid-century Americana, craft cocktails, locally-sourced cuisine
- **Anthony Bourdain connection**: Part of annual Bourdain Birthday Week programming

**Key learning from the exemplar**:
- *Casablanca* as hyperreal cultural node (Eco: "clichés talking among themselves")  
- Rick's Café mythology translated to American bar atmosphere
- Film screening rights + venue licensing as design constraints
- Culinary translation (Moroccan cuisine meeting American bar kitchen capabilities)
- How venue's existing identity (American) paradoxically supports Moroccan/French colonial atmosphere
- Revenue model: ticket sales + F&B uplift + cultural capital (Bourdain legacy preservation)

**This is the METHOD students will apply to their own context**

### Student Application: Design for YOUR Local Venue

**Function**: Students APPLY same ARIES methodology to venues in THEIR cities

**The assignment structure**:
- **Step 1**: Identify 3 potential venues in your city (bars, restaurants, cultural spaces)
- **Step 2**: Analyze each venue's affordances (architecture, existing programming, audience, brand)
- **Step 3**: Select ONE venue for your design
- **Step 4**: Choose cultural icon/event theme appropriate to that venue
- **Step 5**: Apply ARIES methodology to create complete pre-visualization

**Localization is essential**:
- Students in Austin might design BBQ-themed Bourdain events at Franklin Barbecue
- Students in Lyon might design Paul Bocuse tribute dinners at traditional bouchons  
- Students in Montreal might design Leonard Cohen tribute nights at folk music venues
- Students in Vancouver might design events at venues like The American (if not already programmed)

**Students design pre-visualizations showing**:
- How their chosen cultural icon coordinates with venue identity
- How local context (neighborhood, transit, nearby attractions) enables programming
- How licensing complexity (IP rights, music, food/beverage) shapes revenue models
- How event fits into venue's seasonal programming calendar

**The pedagogical progression**:
1. **Week 1-3**: Deep analysis of Casablanca @ The American (learn from exemplar)
2. **Week 4**: Venue research in your city (identify 3 candidates, choose 1)
3. **Week 5-7**: Apply ARIES to your chosen venue/theme (create your own pre-viz)

**This ensures**: Every student works with REAL local context while learning from proven methodology
